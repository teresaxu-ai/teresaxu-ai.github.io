How to Use Claude AI with Power BI: Three Approaches Compared

Claude can connect directly to your Power BI models — reading your tables, writing DAX measures, and in some cases building full report visuals. There are three practical ways to do this and they are not interchangeable.

The honest summary: MCP gives Claude live access to your semantic model. pbi-cli does the same plus gives Claude the ability to build report visuals. Manual chat is the fallback when you cannot or do not want to install anything.

Understanding the Two Layers in Power BI

Before choosing an approach it helps to know that a Power BI file has two distinct layers:

Semantic model layer — the data structure underneath: your tables, relationships, DAX measures, RLS (row-level security) roles, and calculation groups. This is what the MCP and pbi-cli's model tools work with.

Report layer — the visual output: charts, tables, cards, slicers, pages, filters, and themes. This is what only pbi-cli can touch programmatically.

The MCP server only ever works with the semantic model layer. It has no awareness of your report pages or visuals at all.

Method 1

MCP Server

What it is

MCP stands for Model Context Protocol — an open standard that lets Claude connect to external tools through a defined interface. For Power BI, an MCP server acts as a bridge between Claude and your live Power BI Desktop session. Claude sends a request ("create a measure called Total Revenue"), the MCP server translates that into an operation on your model, and the result appears instantly in Power BI Desktop.

There are several MCP server options for Power BI. The two most relevant are:

Microsoft's official Power BI Modeling MCP Server — Microsoft's first-party tool, available as a VSCode extension. It is in public preview (as of mid-2026) and covers tables, columns, measures, relationships, DAX queries, calculation groups, RLS roles, partitions, and hierarchies. Because it is designed primarily for use with GitHub Copilot in VSCode, it works most naturally in that environment. Using it with Claude requires some additional configuration steps.

MCP Engine — a community-built MCP server designed specifically to work well with Claude Desktop and Claude Code. The free tier covers model exploration, DAX query execution, bulk operations, and natural language queries. The Pro tier adds snapshots and rollback, a test runner for validating measures, RLS impersonation testing, impact analysis, and data masking. It connects to Power BI Desktop's local Analysis Services instance and runs entirely on your machine with no telemetry.

For most people using Claude, MCP Engine is the more straightforward starting point. The Microsoft server is the right choice if you are already invested in the VSCode + Copilot workflow or want Microsoft's official support.

What Claude can do via MCP

  • Read your entire model structure — every table, column, measure, relationship, and hierarchy
  • Write new DAX measures and apply them live to your open file
  • Edit or delete existing measures
  • Execute DAX queries and return results within the conversation
  • Create and manage table relationships
  • Set up RLS roles and define row filter expressions
  • Manage calculation groups and calculation items
  • Bulk-rename, audit, or document your model

What Claude cannot do via MCP

It cannot touch the report layer at all. No charts, no pages, no filters, no themes. If you ask Claude via MCP to "add a bar chart," it simply has no tools to do that. The MCP server is a semantic model tool only.

Setup: MCP Engine with Claude Desktop

What you need: Power BI Desktop (Windows), Node.js v18 or later, Claude Desktop.

1

Save your file in .pbip format

Open your file in Power BI Desktop and go to File → Options and settings → Options → Preview features. Enable Power BI Project (.pbip) save format and Store semantic model using TMDL format. Then save via File → Save as and choose the Power BI Project format.

2

Configure Claude Desktop

Claude Desktop stores its MCP configuration at %APPDATA%\Claude\claude_desktop_config.json on Windows. Open the file and add MCP Engine:

{ "mcpServers": { "mcp-engine": { "command": "npx", "args": ["-y", "mcp-engine@latest"] } } }
3

Restart Claude Desktop fully

Closing the window is not enough — the MCP server process needs to be stopped and restarted. On Windows, right-click the Claude icon in the system tray and choose Quit. Then relaunch.

4

Verify the connection

With Power BI Desktop open and your .pbip file loaded, start a new conversation and ask: "Connect to my local Power BI model and list my tables." If Claude returns your actual table names, the connection is working. If it doesn't, check that Node.js is installed and that npx is available in your terminal.

Setup: MCP Engine with Claude Code

Claude Code (available as a VSCode extension or standalone terminal app) uses a different configuration method. Run this once in your terminal:

claude mcp add mcp-engine -- npx -y mcp-engine@latest

Then verify inside a Claude Code session with /mcp — it will show connected servers and available tools.

Example prompts once connected

"List all my tables and describe the relationships between them." "Create a measure called [Total Revenue] in the Sales table: Total Revenue = SUM(Sales[Amount])" "Find all measures that reference columns that no longer exist in the model." "Rename every measure that starts with 'Old_' by removing that prefix. Show me what you plan to change before you apply it." "Create an RLS role called 'RegionManager' that filters the Sales table to rows where Sales[Region] = USERPRINCIPALNAME()." "Run a DAX query to validate that [Revenue YTD] returns correct values for the last three years."
Method 2

pbi-cli

What it is

pbi-cli is an open-source community tool — not an official Microsoft or Anthropic product — built specifically for use with Claude Code. It was created by an individual developer and is actively maintained on GitHub.

It has two layers:

Semantic model layer — connects to Power BI Desktop via direct .NET TOM (Tabular Object Model) interop, the same underlying API that tools like Tabular Editor use. It gives Claude the same model-editing capabilities as the MCP approach: measures, tables, relationships, RLS, snapshots, diagnostics.

Report layer — reads and writes PBIR (Power BI Enhanced Report Format) JSON files directly from your .pbip project folder. This does not require a live connection to Power BI Desktop. Claude edits the report files, and when you reload the file in Desktop the changes appear. This is what enables Claude to actually build visuals.

The report layer is the key thing pbi-cli adds over the MCP. It is the only approach in this comparison where Claude can place a chart on a page without you doing it manually.

What Claude can do via pbi-cli

Everything the MCP can do with the model, plus:

  • Add visuals to report pages — 32 types including bar, line, column, pie, scatter, table, matrix, card, KPI, slicer, and more
  • Bind data fields to visual roles (axes, values, legends, tooltips)
  • Create and delete report pages
  • Set page-level and visual-level filters (categorical, TopN, relative date)
  • Apply report themes
  • Manage bookmarks and drillthrough
  • Snapshot and restore the entire model (TMDL export/import)
  • Run query traces to diagnose slow measures

What it cannot do

It requires your report to be saved in .pbip format. The old .pbix single-file format is not supported for report layer operations. For model-only work (no visuals), .pbix open in Desktop still works via the TOM connection. It also requires Claude Code — the VSCode extension or terminal app — not the claude.ai chat interface.

Setup

What you need: Windows, Python 3.10 or later, Power BI Desktop, Claude Code (VSCode extension or standalone).

1

Save your file as .pbip

Same as for the MCP: File → Save as → Power BI Project in Power BI Desktop. The .pbip format is required because the report layer reads and writes the JSON files inside the project folder directly.

2

Install pbi-cli

The recommended way is with pipx, which installs the tool in its own environment and adds it to your PATH automatically:

pipx install pbi-cli-tool

If you do not have pipx, install it first with pip install pipx and then run pipx ensurepath. Alternatively with standard pip:

pip install pbi-cli-tool

PATH note (Windows): With pip on Windows the pbi command may not be on your PATH. If Windows can't find it, find the Scripts path with python -c "import site; print(site.getusersitepackages().replace('site-packages', 'Scripts'))" and add the output to your system PATH via System → Advanced system settings → Environment variables.

3

Connect to Power BI Desktop

Open Power BI Desktop with your .pbip file loaded, then run pbi connect. This auto-detects the running Power BI Desktop instance, establishes the connection, and installs 12 skill files into your project that teach Claude Code what it can do.

4

Open Claude Code

In VSCode: open the Command Palette (Ctrl+Shift+P) and run "Claude: Open". You do not need to explain what pbi-cli can do — the skill files installed in the previous step give Claude Code that context automatically.

Alternative: skip manual setup entirely. If you would rather let Claude do the work, open Claude Code and say: "Install and set up pbi-cli from https://github.com/MinaSaad1/pbi-cli.git". Claude Code will clone the repo, install the package, run pbi connect, and configure everything.

Example prompts once connected

"Add a clustered bar chart to the Overview page showing Total Revenue by Region." "Create a new page called Executive Summary. Add a card for Total Revenue, a card for Gross Margin %, and a line chart showing revenue trend by month." "Add a TopN filter to the product table visual — top 10 products by Total Revenue." Method 3

Manual / Context in Chat

What it is

No tools, no installation. You bring the context to Claude by pasting your model information into the claude.ai chat, and Claude writes DAX or gives you advice that you apply yourself.

What works well

Writing DAX measures. If you paste your table schema (column names, data types, relationships) into chat and describe what you want, Claude will write accurate DAX using your actual field names. This is reliable and genuinely useful — the main cost is the manual copy-paste step.

Layout and visual design advice. Claude can recommend which chart types suit your KPIs, how to structure your report pages, what slicers and filters to include, and how to arrange visuals for readability. You then build it yourself in Power BI Desktop.

Reviewing and debugging DAX. Paste a broken measure into chat and Claude will diagnose what is wrong and rewrite it.

What does not work well

Generating a working .pbix or .pbip file from scratch. Power BI's file format is complex and not publicly documented in full. Claude can write the individual pieces — measures, table definitions, M queries — but assembling them into a file that Power BI will open reliably is not something that works in practice. Treat any generated file as a rough starting point, not a finished report.

Editing report visual JSON by hand. The report JSON in a .pbip file is structurally complex. Claude can read it and suggest changes, but one incorrect property causes Power BI to reject the entire file on import. This is fragile enough that it is not a recommended workflow — use pbi-cli instead if you need Claude to manage visuals.

When to reach for it

  • You are on macOS or a machine without Power BI Desktop
  • You are in a restricted environment where you cannot install tools
  • You want a quick DAX measure and the copy-paste overhead is acceptable
  • You want Claude to design the report layout before you build it

Which Should You Use?

Use the MCP if your primary goal is model work — writing measures, auditing your schema, setting up security — and you want to do it from claude.ai chat with changes applied live. MCP Engine is the fastest setup; the Microsoft Modeling MCP Server is the right choice if you are already in the VSCode + Copilot ecosystem.

Use pbi-cli if you want Claude to build the report, not just the model underneath it. It is the only approach where Claude can actually place visuals on a page. The setup takes a few minutes and requires Claude Code, but the payoff is a significantly more capable workflow.

Use manual chat as a fallback — or as a complement to the other two when you are away from your Windows machine and just need a measure written.

The MCP and pbi-cli are not mutually exclusive. Use the MCP in claude.ai chat for model refinement during the day, and switch to Claude Code with pbi-cli when you want to build or rearrange report pages.
